# NexusMind 环境配置说明

## 📋 配置文件结构

```
src/main/resources/
├── application.yml          # 主配置文件（公共配置）
├── application-dev.yml      # 开发环境配置
└── application-prod.yml     # 生产环境配置
```

---

## 🔧 环境说明

### 1. 开发环境 (dev)

**激活方式：**
- Maven: `mvn spring-boot:run` (默认)
- 命令行：`java -jar target/NexusMind-0.0.1-SNAPSHOT.jar --spring.profiles.active=dev`
- IDE: 设置 VM 参数 `-Dspring.profiles.active=dev`

**特性：**
- ✅ H2 内存数据库（重启数据清空）
- ✅ H2 控制台访问（/h2-console）
- ✅ Hibernate 自动 DDL（ddl-auto: update）
- ✅ 详细 SQL 日志输出
- ✅ Swagger UI 启用
- ✅ DEBUG 级别日志
- ✅ 文件日志：logs/nexusmind-dev.log

**数据库连接：**
```
URL: jdbc:h2:mem:nexusmind_db
用户名：sa
密码：空
```

**测试账户：**
- 管理员：admin/admin123
- 普通用户：user/user123

---

### 2. 生产环境 (prod)

**激活方式：**
- Maven: `mvn clean package -Pprod && java -jar target/NexusMind-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod`
- Docker: 通过环境变量配置
- 命令行：`java -jar target/NexusMind-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod`

**特性：**
- ✅ PostgreSQL 数据库
- ✅ 禁用 H2 控制台
- ✅ Hibernate DDL 验证（ddl-auto: validate）
- ✅ 精简日志输出
- ✅ 禁用 Swagger UI
- ✅ INFO 级别日志
- ✅ 文件日志：logs/nexusmind-prod.log
- ✅ 连接池优化（HikariCP）
- ✅ Tomcat 性能优化
- ✅ GZIP 压缩启用

**环境变量配置：**

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| DB_HOST | 数据库主机 | localhost | 是 |
| DB_PORT | 数据库端口 | 5432 | 是 |
| DB_NAME | 数据库名称 | nexusmind_db | 是 |
| DB_USERNAME | 数据库用户名 | nexusmind | 是 |
| DB_PASSWORD | 数据库密码 | - | 是 |
| JWT_SECRET | JWT 密钥 | - | 是 |

**示例配置：**
```bash
export DB_HOST=prod-db.example.com
export DB_PORT=5432
export DB_NAME=nexusmind_production
export DB_USERNAME=nexusmind_prod
export DB_PASSWORD=your_secure_password
export JWT_SECRET=your-production-secret-key

java -jar target/NexusMind-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

## 🚀 部署方式

### 方式 1: Maven 命令

**开发环境：**
```bash
mvn spring-boot:run
```

**生产环境：**
```bash
# 1. 使用生产 Profile 打包
mvn clean package -Pprod

# 2. 运行
java -jar target/NexusMind-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

---

### 方式 2: IDE 运行

**IntelliJ IDEA 配置：**

1. 打开 Run/Debug Configurations
2. 选择 NexusMindApplication
3. 在 "Environment variables" 或 "VM options" 中设置：
   - 开发环境：`-Dspring.profiles.active=dev`
   - 生产环境：`-Dspring.profiles.active=prod`

---

### 方式 3: Docker 部署

**Dockerfile 示例：**
```dockerfile
FROM openjdk:21-slim

WORKDIR /app

COPY target/NexusMind-0.0.1-SNAPSHOT.jar app.jar

# 默认使用生产环境
ENV SPRING_PROFILES_ACTIVE=prod
ENV DB_HOST=localhost
ENV DB_PORT=5432
ENV DB_NAME=nexusmind_db
ENV DB_USERNAME=nexusmind
ENV DB_PASSWORD=changeit
ENV JWT_SECRET=change-this-secret-key

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

**构建和运行：**
```bash
# 构建镜像
docker build -t nexusmind:latest .

# 运行容器
docker run -d \
  -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e DB_HOST=postgres \
  -e DB_PORT=5432 \
  -e DB_NAME=nexusmind_db \
  -e DB_USERNAME=nexusmind \
  -e DB_PASSWORD=secure_password \
  -e JWT_SECRET=production_secret_key \
  --name nexusmind \
  nexusmind:latest
```

---

### 方式 4: Docker Compose

**docker-compose.yml 示例：**
```yaml
version: '3.8'

services:
  app:
    image: nexusmind:latest
    container_name: nexusmind-app
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=postgres
      - DB_PORT=5432
      - DB_NAME=nexusmind_db
      - DB_USERNAME=nexusmind
      - DB_PASSWORD=secure_password
      - JWT_SECRET=production_secret_key
    ports:
      - "8080:8080"
    depends_on:
      - postgres
    networks:
      - nexusmind-network

  postgres:
    image: postgres:14
    container_name: nexusmind-db
    environment:
      - POSTGRES_DB=nexusmind_db
      - POSTGRES_USER=nexusmind
      - POSTGRES_PASSWORD=secure_password
    volumes:
      - postgres-data:/var/lib/postgresql/data
    networks:
      - nexusmind-network

volumes:
  postgres-data:

networks:
  nexusmind-network:
    driver: bridge
```

**运行：**
```bash
docker-compose up -d
```

## 📊 环境对比

| 特性 | 开发环境 (dev) | 生产环境 (prod) |
|------|---------------|---------------|

| **JWT 密钥** | 固定测试密钥 | 环境变量 |
| **连接池** | 默认配置 | HikariCP 优化 |
| **Tomcat** | 默认配置 | 性能优化 |
| **GZIP 压缩** | ❌ 禁用 | ✅ 启用 |
| **文件日志** | logs/nexusmind-dev.log | logs/nexusmind-prod.log |

## 🔒 生产环境安全建议

### 1. 数据库安全
- ✅ 使用强密码
- ✅ 限制数据库访问 IP
- ✅ 定期备份数据
- ✅ 使用只读账户运行应用

### 2. JWT 密钥管理
- ✅ 使用环境变量或密钥管理服务
- ✅ 密钥长度至少 32 字符
- ✅ 定期轮换密钥
- ✅ 不要提交到版本控制

### 3. 日志安全
- ✅ 避免记录敏感信息（密码、token）
- ✅ 配置日志轮转
- ✅ 定期清理日志文件

### 4. 网络安全
- ✅ 使用 HTTPS（通过反向代理）
- ✅ 配置防火墙规则
- ✅ 限制 CORS 来源

### 5. 应用安全
- ✅ 定期更新依赖
- ✅ 禁用不必要的端点
- ✅ 配置适当的安全头

## 🔍 故障排查

### 查看日志

**开发环境：**
```bash
tail -f logs/nexusmind-dev.log
```

**生产环境：**
```bash
tail -f logs/nexusmind-prod.log
```

### 检查激活的 Profile

启动日志中会显示：
```
The following 1 profile is active: "dev"
```

或在应用中访问：
```
GET /actuator/env
```

### 数据库连接问题

**开发环境：** 访问 H2 控制台检查
```
URL: http://localhost:8080/h2-console
JDBC URL: jdbc:h2:mem:nexusmind_db
```

**生产环境：** 检查环境变量
```bash
echo $DB_HOST
echo $DB_PASSWORD
```
